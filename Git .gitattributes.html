Git .gitattributes

গিট কীভাবে ফাইলগুলি পরিচালনা করে তা নিয়ন্ত্রণ করুন

.gitattributes কি?

একটি .gitattributes ফাইল হল একটি বিশেষ ফাইল যা গিটকে বলে যে কীভাবে আপনার সংগ্রহস্থলে নির্দিষ্ট ফাইলগুলি পরিচালনা করতে হয়।

এটি লাইন এন্ডিং, ফাইলের ধরন, মার্জ আচরণ, কাস্টম ডিফ টুল এবং আরও অনেক কিছু নিয়ন্ত্রণ করে।

যেহেতু এই ফাইলটি আপনার প্রোজেক্টের সাথে ভার্সন করা হয়েছে, তাই আপনার গ্রুপের সবাই একই সেটিংস পায়।

ট্যাক্স ফলাফল

বিভিন্ন অপারেটিং সিস্টেম জুড়ে সামঞ্জস্যপূর্ণ লাইন ফলাফল বাস্তবায়ন করা

বাইনারি ফাইল

ফাইলগুলিকে বাইনারি হিসাবে চিহ্নিত করতে (গিট তাদের মার্জ বা প্রতিস্থাপন করার চেষ্টা করবে না)

Git LFS

বড় ফাইলের জন্য Git LFS সক্ষম করতে

কাস্টম সরঞ্জাম

বিশেষ ফাইল প্রকারের জন্য কাস্টম ডিফ বা মার্জ টুল সেট আপ করুন

1

.gitattributes তৈরি বা সম্পাদনা করুন

আপনার সংগ্রহস্থলের মূলে নেভিগেট করুন (বা স্থানীয় নিয়মের জন্য একটি সাবডিরেক্টরি)।

.gitattributes ফাইলটি তৈরি বা সম্পাদনা করুন।

গিট কীভাবে ফাইলগুলিকে একবারে একটি লাইন পরিচালনা করবে তার নিয়ম যুক্ত করুন।

উদাহরণ: সমস্ত টেক্সট ফাইলের জন্য জোর করে ইউনিক্স লাইন শেষ করুন

*.txt text eol=lf
2

ট্যাক্স ফলাফল হ্যান্ডেল

বিভিন্ন OS এ মার্জ দ্বন্দ্ব এবং ভাঙা ফাইলগুলি এড়াতে লাইনের ফলাফলকে মানসম্মত করুন।

উদাহরণ: শেল স্ক্রিপ্টের জন্য LF সেট করুন

*.sh text eol=lf
3

ফাইলগুলিকে বাইনারি হিসাবে চিহ্নিত করুন

গিটকে বলুন কোন ফাইলগুলি বাইনারি (টেক্সট নয়)।

এটি এই ফাইলগুলির জন্য লাইনের ফলাফলগুলিকে একত্রিত করতে বা পরিবর্তন করার চেষ্টা করতে গিটকে বাধা দেয়।

উদাহরণ: PNG ফাইলগুলিকে বাইনারি হিসাবে চিহ্নিত করুন

*.png binary
4

ফাইল প্রকারের জন্য LFS সক্ষম করুন

ছবি বা ডেটাসেটের মতো বড় ফাইলের জন্য Git LFS ব্যবহার করুন।

এটি গিটকে এই ফাইলগুলির জন্য LFS ব্যবহার করতে বলে:

উদাহরণ: LFS দিয়ে PSD ফাইল ট্র্যাক করুন

*.psd filter=lfs diff=lfs merge=lfs -text
5

কাস্টম ডিফ সেটিংস

নির্দিষ্ট ফাইলের ধরন (যেমন মার্কডাউন বা জুপিটার নোটবুক) তুলনা করার জন্য গিটকে একটি বিশেষ টুল ব্যবহার করতে বলুন:

উদাহরণ: মার্কডাউনের জন্য কাস্টম ডিফ

*.md diff=markdown
6

গুণাবলী পরীক্ষা করুন

একটি ফাইলের জন্য কি বৈশিষ্ট্য সেট করা আছে দেখুন:

উদাহরণ: একটি ফাইলের বৈশিষ্ট্য পরীক্ষা করুন

git check-attr --all README.md

কৌশলগুলি একত্রিত করুন

জটিল ফাইলগুলির জন্য কাস্টম মার্জ ড্রাইভার সেট আপ করুন (যেমন লক ফাইল বা নোটবুক)।

Export-ignore

git archiveউত্পন্ন টার/জিপ সংরক্ষণাগার থেকে ফাইলগুলি বাদ দিন:

docs/* export-ignore

টিপস এবং সর্বোত্তম অনুশীলন

1

Patterns

প্যাটার্ন .gitignore (ওয়াইল্ডকার্ড ইত্যাদি) এর মত কাজ করে।

2

সাবডিরেক্টরি

শুধুমাত্র সেখানে প্রযোজ্য নিয়মগুলির জন্য সাবডিরেক্টরিতে .gitattributes রাখুন।

3

রেট্রোস্পেকটিভ এটা ঠিক করবে না

.gitattributes পরিবর্তন করা পূর্ববর্তীভাবে ইতিমধ্যে প্রতিশ্রুতিবদ্ধ ফাইলগুলিকে ঠিক করে না—সেগুলিকে আপডেট করতে ফাইলগুলিকে আবার যুক্ত করুন৷
